# LoungeFeed website placeholders

Replace these before publishing the public website or submitting the Pinterest application.

## Required before public publishing

### `[SITE_URL]`

The final public URL for the LoungeFeed site.

Temporary GitHub Pages example:

```text
https://loungefeed.github.io/
```

Future custom-domain example:

```text
https://loungefeed.app/
```

### `[SUPPORT_EMAIL]`

Use a product-specific support email, not a personal email.

Good examples:

```text
support@loungefeed.app
hello@loungefeed.app
loungefeed@outlook.com
```

### `[PUBLIC_OPERATOR_NAME]`

The name shown publicly on the website and privacy policy.

Use the least personal accurate public identity available.

Possible temporary value:

```text
LoungeFeed Project
```

Future value after a company/studio exists:

```text
<Company or studio legal name>
```

Do not publish a personal name if the goal is to reduce personal traceability.

## Required for private platform forms

These may be required inside GitHub, Pinterest, Apple, domain registrar, or payment forms. They do not need to be shown publicly on the LoungeFeed website unless the platform requires it.

- Account owner legal name
- Billing details
- Country/region required by the platform
- Tax details if monetization or paid developer programs require them

Use truthful information in private platform forms. The public website can still use the product brand as the visible operator name when legally appropriate.

## Optional later

### `[APP_STORE_URL]`

Only add after the Apple TV app has an App Store page.

### `[PRESS_EMAIL]`

Only add if a separate media/contact email exists.

### `[COMPANY_ADDRESS]`

Do not add a personal home address. Add only if a legal company/studio address is available and required.
