How LoungeFeed works

LoungeFeed separates viewing from account access. The Apple TV app stays focused on watching. A local Host app on the user's Mac or iPhone handles sign-in, pairing, local caching, and source-specific API connections.

Local Host

The Host runs on a device the user controls. It connects to authorized sources and serves content only to the paired Apple TV viewer on the local network.

Read-only by design

LoungeFeed is for viewing. It does not post, like, follow, comment, save, or otherwise perform social-writing actions on behalf of the user.

TV-first browsing

The viewer is optimized for a remote control, large media, replies/messages, and full-screen photo and video playback.

Pinterest integration

LoungeFeed is preparing a Pinterest integration that lets users view their own authorized Pinterest boards and pins on Apple TV.

The planned Pinterest integration is read-only and uses Pinterest OAuth. LoungeFeed will request only the minimum read scopes needed to show a user's boards, pins, and account identity. It will not collect Pinterest passwords, scrape Pinterest web pages, use Pinterest session cookies, create pins, save pins, follow accounts, or modify the user's Pinterest account.

Privacy basics

No password collection

Users sign in through each source's own authorization flow. LoungeFeed does not ask users to type their source passwords into LoungeFeed.

Local tokens

Source tokens and local cache data are stored on the user's Host device, not on a LoungeFeed cloud service.

User control

Users can disconnect a source account and clear local cached data from the Host app.
